Why use a modular CSS Philosophy?

  • Your CSS with have a high level of specificity while remaining highly reusable.
  • You (should) never have to rewrite code. Potentially, share code across different projects.
  • You will avoid CSS conflicts and changes breaking other areas of your site.

Controlling Drupal Markup

Module: Fences

Enable class overrrides

Select HTML5 element on a per field basis

- You can configure fences to use one class on a per field basis. Keeps you field name consistent whether it is rendered by DS, Views field or Drupal core. - On a per field basis, you will also be able to define with HTML5 element you would liek to use. - I always prefer building my displays with DS becasue it is reusable. But sometime find myself needed views funtionality that only works with field displays. This will keep those two views consistent.
